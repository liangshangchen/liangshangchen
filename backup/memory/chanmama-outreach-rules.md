# 蝉妈妈达人外联 — 执行规则手册

---

## 一、总流程

1. 打开蝉妈妈达人榜
2. 进入指定分类
3. 确认选中「达人号」（不能误用品牌自营号）
4. 按粉丝门槛筛选达人
5. 从列表逐个找符合条件的人
6. 打开达人详情页，提取资料
7. 跳转抖音主页
8. 处理登录/验证码
9. 打开私信窗口
10. 输入消息但不自动发送
11. 记录结果并输出报告

---

## 二、筛选规则

- 必须在**蝉妈妈达人榜**执行
- 必须进入指定类目（当前重点：**医药保健 → 保健品、滋补品**）
- 必须真正勾选**「达人号」**
- 按粉丝门槛筛选（如 20w+ → `--min-fans 200000`）
- 多个符合条件时按序号选取（`--match-index 1 / 2 / 3...`）

---

## 三、达人选择规则

- 只处理粉丝数达到门槛的达人
- 按列表顺序找第 N 个符合条件的人
- **已联系过的达人自动跳过**
- 通过 state 文件去重，避免重复骚扰

---

## 四、翻页规则

- 当前页不够时，自动检测分页组件继续翻页
- 逐页继续遍历
- 从详情页/抖音页返回异常时：
  - 先尝试 `go_back`
  - 不行则回达人列表重新定位
- **目标**：不中断地继续找下一个达人

---

## 五、信息提取规则

### 蝉妈妈详情页提取
昵称、抖音号、粉丝数、活跃粉丝、MCN、简介、联系方式、抖音主页链接

### 抖音主页提取
个人简介、简介中的联系方式、私信入口

### 合并处理
- 两端数据合并去重
- trim 空格
- 标准化联系方式格式

---

## 六、联系方式识别规则

不限于「微信号」三个字，覆盖常见隐晦写法：

**微信/商务号类**

| 关键词 | 示例 |
|--------|------|
| 微信、微信号 | 微信 abc123 |
| wx / WX | wx：hello666 |
| vx / VX / V / v | vx：hello666 |
| 薇信、威信、胃信、薇芯 | 薇信 xxxxx |
| v信 | v信 abc123 |
| 商务、商合、BD | 商合：GGCM0016 |
| 加我、➕、＋ | ➕ abc123 |

**其他类型**
- QQ
- 邮箱
- 手机号
- 🈴后面的合作号

**实际可识别示例**
```
V abc123
vx：hello666
商合：GGCM0016
薇信 xxxxx
910682558@qq.com
13812345678
```

---

## 七、验证码/登录处理规则

**三层降级机制：**

| 层级 | 方式 | 说明 |
|------|------|------|
| 第一层 | 自动识别 | 检测验证码 iframe → 模板匹配 → 自动拖滑块 |
| 第二层 | 插件兜底 | 本地自动解失败 → CapSolver 插件尝试 |
| 第三层 | 人工处理 | 全部失败 → 等待手动过验证 → 脚本自动继续 |

- 有超时控制，不会无限卡死
- 人工过完验证后无需重跑，脚本自动恢复

---

## 八、私信规则

- 只负责**打开私信窗口**
- 只负责**把消息输入进去**
- **默认不自动发送**
- 页面停留等待人工确认
- 最终发不发，**由人拍板**

---

## 九、去重与断点续跑

- 已联系达人记入 state 文件
- 下次运行自动跳过已处理的
- 中断后可接着跑
- 不会把同一个抖音号反复当新达人处理

---

## 十、输出规则

**报告内容：**
- 达人明细
- 成功/失败统计
- 联系方式汇总
- 粉丝量排行
- 失败原因

**输出形式：**
- Markdown 报告
- XLSX 报告
- 可自动发飞书

---

## 十一、脚本选择规则

| 场景 | 脚本 | 适用 |
|------|------|------|
| 单个达人/单次流程 | `run_workflow_v6.py` | 找第 N 个符合条件达人、验证流程 |
| 批量跑达人 | `batch_outreach.py` | 自动遍历、翻页、连续处理、完整报告 |

---

## 十二、硬规则（不可变更）

1. 必须在现有 Chrome 会话里跑
2. 必须优先复用已登录状态
3. 必须确认是达人号
4. 默认只输入私信，不直接发送
5. 验证码失败要允许人工接管
6. 已联系过的达人必须跳过
7. 每次处理都要留结果记录

---

## 十三、一句话总结

> 先在蝉妈妈里按类目 + 达人号 + 粉丝门槛找达人，逐个进详情提取资料，再跳抖音开私信，遇到验证码先自动解、解不了就等手动，最后只帮把消息准备好，不擅自发送，并把所有处理结果记进报告。
