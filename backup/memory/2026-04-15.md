### 3. Hermes Agent 学习笔记（来源：知乎文章）

#### 是什么
- **Hermes Agent**：Nous Research 打造的开源、自托管 AI Agent
- 核心定位：**自主智能体**，不是 IDE 编程助手，也不是套壳聊天机器人
- **最大亮点：内置学习循环机制** — 记住你教的东西，下次自动用学会的方法解决，还会自己创建"技能手册"
- 部署在本地服务器，**运行时间越长能力越强**（自我进化）

#### 核心特性
- **一行命令安装**：`curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash`
- Windows 支持 WSL2 或 PowerShell 安装：`irm ... | iex`
- **支持 Claude 模型**（Anthropic 已关闭 OpenClaw 的 Claude 订阅入口，Hermes 还能用）
- 支持的提供商：Anthropic、OpenAI Codex、Qwen、OpenRouter、Hugging Face、GitHub Copilot、MiniMax
- **28 个内置工具** + **105 个 Skills**
- 终端 TUI 界面，ASCII art 风格
- 支持从 OpenClaw 迁移配置（`hermes claw migrate`）
- 支持 Daytona/Modal 无服务器部署（闲置不收费）
- 有 RL 训练框架和轨迹导出（深度研究 AI 用）

#### 基本命令
- `hermes` — 启动交互式对话
- `hermes model` — 选择模型提供商和模型
- `hermes tools` — 配置启用哪些工具
- `hermes config set` — 设置单个配置项
- `hermes gateway` — 启动消息网关（Telegram、Discord 等）
- `hermes setup` — 重新运行完整配置向导
- `hermes claw migrate` — 从 OpenClaw 迁移配置
- `hermes update` — 更新到最新版本
- `hermes doctor` — 诊断问题

#### 消息通道
- 支持：Telegram、Discord、Webhooks 等
- **不直接支持飞书和微信**（可能通过 Webhooks 连接飞书）

#### 与 OpenClaw 的对比
| 维度 | OpenClaw | Hermes Agent |
|------|----------|-------------|
| 定位 | 个人 AI 工作台 + 消息中枢 | 自进化 AI 数字分身 |
| 飞书支持 | ✅ 深度集成 | ❌ 需 Webhooks |
| Claude 支持 | ❌ 已被关闭 | ✅ 可用 |
| 无服务器 | ❌ | ✅ Daytona/Modal |
| RL 训练 | ❌ | ✅ |
| 自学习循环 | ❌ | ✅ 核心亮点 |
| 子 Agent 协调 | ✅ | 待确认 |
| 团队协作 | 基础 | DM pairing + 多用户授权更成熟 |
| MCP Server | 可作为 ACP 客户端 | 可作为 MCP Server 被 OpenClaw 调用 |

#### 互补方案
- Hermes 可作为 **MCP Server**，让 OpenClaw 通过 ACP 调用它
- 两个系统可同时运行、协同工作
